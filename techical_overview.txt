Technical Design Overview:

Goal:
Build a media provenance system with two core capabilities:
1) Certification: register original media before publication.
2) Verification: check if later media matches a certified original.

We use a two-layer trust model:
- SHA-256 hash for exact file integrity (detects any byte change).
- Ed25519 signature for authenticity (proves certified issuer signed the record).
This is simple, secure, low-cost, and realistic for our time frame.

How the system works:
1) Registration flow (implemented first)
- User uploads original file.
- Backend computes SHA-256 of raw file bytes.
- Backend creates a manifest (file metadata + hash).
- Backend signs manifest with Ed25519 private key.
- Store manifest, signature, key_id, and hash in database.
- Return proof package (asset id, manifest, signature, public key reference).

2) Verification flow (core)
- User uploads suspect file.
- Backend computes SHA-256.
- If hash matches certified record, verify Ed25519 signature.
- If both pass: Authentic Original.
- If hash not found: Not Found (fallback similarity can be added later).

This project has a big flaw. When uploading a file to any social media platform, 
it's almost guaranteed that their codec will compress or alter the file in a way
that causes a hash mismatch, meaning that a file downloaded from a website will 
likely not match the original and thus be flagged as inauthentic, even if it 
otherwise has been unaltered. 
There are many solutions for this, but none that we can achieve with our resources, 
so our best bet would be to mention the issue to the judges, and note how we would 
solve it given more resources. I could also implement a very weak and rudimentary
solution to the issue just to prove that something can be done.

Languages and frameworks
- Python 3.12: core cryptography + hashing + API logic.
- FastAPI: lightweight backend API for registration/verification.
- PostgreSQL: persistent storage for manifests and keys.
- Frontend stack: Next.js + TypeScript for certification and public portal UI.

Libraries in use
- hashlib (standard library): SHA-256 hashing.
- PyNaCl (nacl.signing): Ed25519 key generation, signing, verification.
- json (standard library): canonical manifest serialization.
- psycopg (or psycopg2): PostgreSQL connection and inserts/queries.
